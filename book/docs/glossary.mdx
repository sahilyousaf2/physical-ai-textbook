# Glossary of Robotics and AI Terminology

This glossary provides definitions for key terms used throughout the Physical AI & Humanoid Robotics textbook. Understanding these terms is essential for mastering the concepts covered in this course.

## A

**Action Space**: The set of all possible actions that an agent can take in a given environment.

**Affordance**: The possibility of an action on an object or environment feature, representing what can be done with it.

**Artificial Intelligence (AI)**: The simulation of human intelligence processes by machines, especially computer systems.

## B

**Behavior Tree**: A hierarchical structure used in robotics and game AI to organize and execute complex behaviors.

**Bipedal Locomotion**: The act of walking on two legs, a key capability for humanoid robots.

**Bridge (ROS)**: A communication layer that enables data exchange between ROS and other frameworks (e.g., Isaac ROS bridge).

## C

**Capture Point**: A location where a robot should step to stop safely, used in balance control for bipedal robots.

**Center of Mass (CoM)**: The point where the total mass of a body may be assumed to be concentrated for the purpose of calculations.

**Cognitive Planning**: The process of using high-level reasoning to create plans for achieving goals.

**Computer Vision**: A field of AI that trains computers to interpret and understand the visual world.

## D

**Deep Learning**: A subset of machine learning based on artificial neural networks with representation learning.

**Deep Reinforcement Learning**: A type of machine learning that combines deep learning with reinforcement learning.

**Domain Randomization**: A technique for training AI models by randomizing simulation parameters to improve real-world transfer.

**Dynamic Walking**: A walking style that involves controlled falling, allowing for faster and more human-like locomotion.

## E

**Embodied AI**: Artificial intelligence that is integrated with a physical body and interacts with the real world.

**End-Effector**: The device at the end of a robotic arm that interacts with the environment (e.g., gripper, tool).

**Environment (in RL)**: The external system with which an agent interacts, providing observations and rewards.

## F

**Forward Kinematics**: The process of determining the position and orientation of the end-effector given joint angles.

**Framework**: A foundational structure that provides functionality for developing applications in a specific domain.

## G

**Gazebo**: A physics-based 3D simulation environment used for robotics applications.

**Generalization**: The ability of a model to perform well on unseen data or in new environments.

**GPU Acceleration**: The use of graphics processing units to speed up computational tasks, especially in AI.

## H

**Human-Robot Interaction (HRI)**: The study of interactions between humans and robots.

**Hyperparameter**: A parameter whose value is set before the learning process begins, controlling the learning process.

## I

**Inverted Pendulum**: A model used in robotics to represent balance control, where a mass is balanced on a moving base.

**Inverse Kinematics**: The process of determining joint angles required to achieve a desired end-effector position.

**Isaac Sim**: NVIDIA's simulation environment for robotics development with photorealistic rendering.

**Isaac ROS**: NVIDIA's collection of hardware acceleration packages for ROS 2.

## K

**Kinematics**: The study of motion without considering the forces that cause the motion.

**Kinesthetic Teaching**: A method of programming robots by physically guiding them through motions.

## L

**Large Language Model (LLM)**: An AI model with many parameters trained on large text datasets, capable of natural language understanding and generation.

**LiDAR**: Light Detection and Ranging, a remote sensing method using light in the form of a pulsed laser.

**Linear Inverted Pendulum (LIP)**: A simplified model of balance control assuming constant height.

**Locomotion**: The ability to move from one place to another, particularly relevant for mobile robots.

## M

**Manipulation**: The ability to physically interact with objects in the environment, typically using robotic arms.

**Markov Decision Process (MDP)**: A mathematical framework for modeling decision-making in situations with probabilistic outcomes.

**Machine Learning**: A subset of AI that provides systems the ability to automatically learn and improve from experience.

**Middleware**: Software that provides common services and capabilities to applications beyond what's offered by the operating system.

## N

**Navigation**: The process of planning and executing movement from one location to another.

**Nav2**: The navigation stack for ROS 2, providing path planning and obstacle avoidance.

**Neural Network**: A computing system inspired by the human brain, used for machine learning tasks.

**Node**: A process that performs computation in a ROS system.

## O

**Obstacle Avoidance**: The capability to detect and navigate around obstacles in the environment.

**Omnidirectional**: The ability to move in any direction, not constrained to forward/backward and rotation.

**OpenAI Whisper**: A speech recognition model developed by OpenAI for converting speech to text.

## P

**Path Planning**: The computational problem of finding a valid sequence of configurations to move an object from a source to a destination.

**Perception**: The process of interpreting sensory information to understand the environment.

**Physics Engine**: Software that simulates the behavior of physical objects using mathematical models.

**Point Cloud**: A set of data points in space, typically representing the external surface of an object.

**Pose**: The position and orientation of a robot or object in space.

**Probabilistic Robotics**: An approach to robotics that explicitly accounts for uncertainty in sensing and control.

## Q

**Q-Learning**: A model-free reinforcement learning algorithm that learns a policy telling an agent what action to take under what circumstances.

**Quality of Service (QoS)**: Policies in ROS 2 that allow for fine-tuning of communication between nodes.

## R

**Reinforcement Learning**: A type of machine learning where an agent learns to make decisions by taking actions in an environment to maximize cumulative reward.

**Robot Operating System (ROS)**: A flexible framework for writing robot software, providing hardware abstraction, device drivers, and more.

**ROS 2**: The second generation of the Robot Operating System with improved architecture and features.

**ROS Middleware (RMW)**: The abstraction layer between ROS client libraries and DDS implementations.

**Rapidly-exploring Random Tree (RRT)**: A path planning algorithm that builds a tree of possible paths.

## S

**Sensor Fusion**: The process of combining sensory data from multiple sources to improve understanding.

**Simultaneous Localization and Mapping (SLAM)**: The computational problem of constructing or updating a map of an unknown environment while simultaneously keeping track of an agent's location.

**Simulation**: The imitation of the operation of a real-world process or system over time.

**State Space**: The set of all possible states that a system can be in.

**Static Balance**: Balance maintained without movement, typically requiring the center of mass to be over the support base.

## T

**Task Planning**: The process of creating a sequence of actions to achieve a specified goal.

**Tensor**: A multi-dimensional array used in machine learning and AI computations.

**Trajectory**: A time-ordered sequence of positions, orientations, and associated velocities and accelerations.

**Transformer**: A type of neural network architecture that uses attention mechanisms, widely used in natural language processing.

## V

**Visual SLAM (VSLAM)**: SLAM that uses visual sensors (cameras) as the primary input for mapping and localization.

**Voice Command Processing**: The system of recognizing and interpreting spoken commands for robot control.

**Vision-Language-Action (VLA)**: Systems that combine visual perception, language understanding, and physical action.

## W

**World Model**: An internal representation of the environment that an agent uses for planning and decision-making.

**Waypoint**: A reference point used for navigation, typically a coordinate in space.

## Z

**Zero Moment Point (ZMP)**: A point where the ground reaction forces create no moment, used in bipedal balance control.

---

*This glossary will continue to expand as new concepts are introduced in the field of robotics and AI.*
